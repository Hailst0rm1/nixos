# Definition of done — project-idea

Grade the run against every item. Fix anything failing before stopping. An item may be marked done only if it genuinely holds; a cut must be logged in `build-log.md` with a reason.

- [ ] Every shipped idea is backed by live URLs on its Pain, Market, and Competitor evidence — no invented facts, inferences labelled as inferences.
- [ ] The skeptic pass ran and every surviving idea's central claim withstood it.
- [ ] Merit held as the hard filter — no filler idea shipped to pad the count.
- [ ] Diversity soft-constraint met (no profile tag over the cap) — or the shortfall is logged.
- [ ] `ideas.json` validates against the schema and every idea carries evidence URLs.
- [ ] `slides.html` opens in the browser, has the overview slide plus one slide per idea, and each idea slide shows its business goal and signal scorecard.
- [ ] `build-log.md` records every self-answered question and every cut idea with its reason.
- [ ] The output path was printed for the user.
