# Deliverables

## Core — every run

| Deliverable | Built with | Note |
|-------------|-----------|------|
| Brand guidelines (`DESIGN.md`) | `impeccable` (+ `/imagegen` logo) | Complete enough for a stranger to extend |
| Landing page (demand test) | `impeccable` + `copywriting` + `cro` + `offers` + `launch` | Fake-door CTA; screenshot mobile + desktop |

| Red-team | `marketing-council` | Surviving objections go in the recap |
| `recap.html` | `visual-explainer` | Business in 5 min; deliverables map; links resolve |

## Optional — ship 1 or 2 by fit

Pick the highest-leverage extras for *this* business. Quality beats quantity — one polished, verified artifact beats three rushed ones.

| Deliverable | Built with | When it fits |
|-------------|-----------|--------------|
| Ad creatives (top channel) | `ad-creative` | There's an obvious paid acquisition channel |
| Onboarding / email sequence | `onboarding`, `emails` | Activation matters to the model |
| Social profile assets + first-week posts | `social` | The channel is community/social-led |
| Pricing tiers | `pricing` | Self-serve SaaS where packaging does the work |
| Full GTM plan (AARRR) | `marketing-plan` | The user wants a launch-ready operating plan |

## Invent one unexpected deliverable

If a real founder launching this would make something the list above misses, and the available tools can make it, make it — and add it to the recap's deliverables map. One is enough.
